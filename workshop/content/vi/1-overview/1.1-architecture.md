## Kiến trúc Agent-as-Tool

Mỗi trader là một Strands Trading Agent. Trading Agent đăng ký một Research Agent bằng
`researcher.as_tool(name="research_agent")`; Research Agent sử dụng research tools và trả kết quả về cho
Trading Agent trước khi agent quyết định giao dịch.

### Các khối chính

1. **Trading Agent** — quản lý tài khoản, lấy giá, mua/bán, đổi strategy và gửi Telegram.
2. **Research Agent as Tool** — nhận một research objective từ Trading Agent và thực hiện nghiên cứu chuyên biệt.
3. **Research MCP Server** — cung cấp `search_financial_news()` cho Research Agent; MCP Server không chứa agent.
4. **Amazon Bedrock** — cung cấp foundation model cho cả Trading Agent và Research Agent khi dùng Bedrock provider.
5. **DynamoDB** — lưu portfolio state và logs khi `STORAGE_BACKEND=dynamodb`.

### Luồng một trading cycle

```text
Trading Agent
      │
      ├── research_agent(input)
      │       └── Research Agent ──→ search_financial_news()
      │
      ├── account_report() / lookup_share_price()
      ├── buy_shares() / sell_shares() / change_strategy()
      └── send_telegram_message()
```

{{% notice info %}}
Sơ đồ dùng một trader để tránh lặp lại cùng cấu trúc. Runtime thực tế có thể chạy một trader hoặc toàn bộ
bốn trader; mỗi trader tạo Research Agent riêng và sử dụng nó như một tool.
{{% /notice %}}
