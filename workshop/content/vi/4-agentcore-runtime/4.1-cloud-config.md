## Tạo DynamoDB table

Project dùng single-table schema với partition key `pk` và sort key `sk`:

```powershell
aws dynamodb create-table `
  --table-name strands-trader-state `
  --attribute-definitions AttributeName=pk,AttributeType=S AttributeName=sk,AttributeType=S `
  --key-schema AttributeName=pk,KeyType=HASH AttributeName=sk,KeyType=RANGE `
  --billing-mode PAY_PER_REQUEST `
  --region us-west-2

aws dynamodb wait table-exists `
  --table-name strands-trader-state `
  --region us-west-2
```

### Cấu hình Runtime

Tạo file `.env.agentcore` từ mẫu:

```powershell
Copy-Item deploy/agentcore/.env.example .env.agentcore
```

Các biến quan trọng:

```dotenv
MODEL_PROVIDER=bedrock
MODEL_ID=us.anthropic.claude-sonnet-4-20250514-v1:0
AWS_REGION=us-west-2
STORAGE_BACKEND=dynamodb
DYNAMODB_TABLE=strands-trader-state
```

`MODEL_ID` chỉ là ví dụ. Hãy dùng inference profile/model đã được bật trong account và Region của bạn.

### Runtime execution role

Role của Runtime cần tối thiểu:

- `bedrock:InvokeModel` và `bedrock:InvokeModelWithResponseStream` cho model đã chọn.
- `dynamodb:GetItem`, `PutItem`, `Query`, `UpdateItem` trên table workshop.
- Quyền ghi CloudWatch Logs/trace do AgentCore yêu cầu.

{{% notice warning %}}
Trong workshop có thể để AgentCore CLI tạo role. Với môi trường thật, hãy thay bằng role least-privilege
và giới hạn resource ARN cụ thể.
{{% /notice %}}
