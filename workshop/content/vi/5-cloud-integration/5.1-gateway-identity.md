## Vì sao cần Gateway

Ở local, tools gọi Massive, Tavily và Telegram trực tiếp. Trong kiến trúc AWS, AgentCore Gateway tạo một
MCP endpoint được quản trị và đặt Lambda tools phía sau. AgentCore Identity giữ API key ngoài source code.

### Tạo credential providers

Từ project `deploy/agentcore/TraderFloor`:

```powershell
agentcore add credential --name massive-api --api-key $env:MASSIVE_API_KEY
agentcore add credential --name tavily-api --api-key $env:TAVILY_API_KEY
agentcore add credential --name telegram-token --api-key $env:TELEGRAM_BOT_TOKEN
```

Sensitive values được lưu trong `agentcore/.env.local` khi phát triển và được Identity quản lý khi deploy.

### Tạo Gateway

Trong workshop có thể bắt đầu không có inbound authorizer để kiểm tra kết nối nội bộ:

```powershell
agentcore add gateway `
  --name TraderToolsGateway `
  --authorizer-type NONE `
  --runtimes TraderFloor
```

Sau khi Lambda tool và `tools.json` đã sẵn sàng:

```powershell
agentcore add gateway-target `
  --name TradingTools `
  --type lambda-function-arn `
  --lambda-arn <TRADING_TOOL_LAMBDA_ARN> `
  --tool-schema-file tools.json `
  --gateway TraderToolsGateway

agentcore deploy
```

### Production hardening

- Chuyển Gateway sang `CUSTOM_JWT` với Cognito discovery URL và app client audience.
- Scope Runtime role chỉ đến credential provider ARN được phép.
- Tách credential Massive, Tavily và Telegram để rotate độc lập.
- Không trả API key trong tool response hoặc log.

{{% notice warning %}}
`authorizer-type NONE` chỉ dành cho bước thử nghiệm. Không dùng cấu hình này cho endpoint có thể truy cập
từ bên ngoài.
{{% /notice %}}

Tài liệu: [AgentCore Gateway quickstart](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/gateway-quick-start.html)
và [AgentCore Identity credential providers](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/resource-providers.html).
