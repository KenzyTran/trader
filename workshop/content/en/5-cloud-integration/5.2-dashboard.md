## Distribute the dashboard

The frontend in `frontend/` is a static TypeScript/Vite application. Build it for production:

```powershell
cd frontend
npm ci
npm run build
```

Upload `dist/` to a private S3 bucket and place CloudFront in front of it. Do not enable public website hosting directly on the bucket.

### Read-only API

The dashboard only needs these endpoints:

```text
GET /api/market
GET /api/traders
GET /api/traders/{name}
GET /api/traders/{name}/logs
```

In the target architecture, API Gateway invokes a Lambda adapter that reads DynamoDB. Do not allow the dashboard to call buy/sell tools or invoke the Runtime directly.

### Authentication

1. Create a Cognito User Pool and an App Client without a client secret for the browser.
2. Configure an API Gateway JWT authorizer.
3. Route CloudFront `/api/*` requests to API Gateway and all other paths to S3.
4. Allow only `GET` methods on the dashboard API.

{{% notice info %}}
If the workshop focuses only on AgentCore Runtime, you can continue running the dashboard locally and point the local API to DynamoDB. CloudFront and Cognito are extensions, not requirements for completing the Runtime lab.
{{% /notice %}}
