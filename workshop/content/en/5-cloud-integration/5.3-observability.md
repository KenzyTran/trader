## Enable and verify observability

AgentCore Observability sends Runtime and Gateway traces, metrics, and logs to CloudWatch. CloudWatch Transaction Search must be enabled before viewing GenAI traces.

After deployment:

```powershell
agentcore logs
agentcore traces list
agentcore status
```

In CloudWatch, verify:

- Runtime invocations and latency.
- Model invocation spans and token usage.
- Tool-call spans, errors, and execution time.
- Gateway metrics and logs.
- DynamoDB throttling and Lambda errors if the application API is enabled.

### Do not log sensitive data

- Redact API keys, bot tokens, and Authorization headers.
- Do not record complete user financial responses unless necessary.
- Set retention for workshop log groups.
- Retain only prompt and tool payloads needed for debugging.

Reference: [Configure AgentCore observability](https://docs.aws.amazon.com/bedrock-agentcore/latest/devguide/observability-configure.html).
