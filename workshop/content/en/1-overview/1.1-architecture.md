## Agent-as-Tool architecture

Each trader is a Strands Trading Agent. The Trading Agent registers a Research Agent through `researcher.as_tool(name="research_agent")`. The Research Agent uses research tools and returns its findings before the Trading Agent makes a trading decision.

![Strands Agent-as-Tool architecture on Amazon Bedrock AgentCore](../../../images/strands-trader-agentcore.svg)

*One trader represents the repeated pattern: the Trading Agent calls the Research Agent as a tool, while the MCP server only provides research tools.*

**Editable architecture source:** [Download the draw.io file](../../../images/strands-trader-agentcore.drawio)

### Main components

1. **Trading Agent** — manages the account, retrieves prices, buys or sells shares, changes strategy, and sends Telegram messages.
2. **Research Agent as Tool** — receives a research objective from the Trading Agent and performs specialized research.
3. **Research MCP Server** — provides `search_financial_news()` to the Research Agent; the MCP server does not contain an agent.
4. **Amazon Bedrock** — provides the foundation model for both agents when the Bedrock provider is selected.
5. **DynamoDB** — stores portfolio state and logs when `STORAGE_BACKEND=dynamodb`.

### One trading-cycle flow

```text
Trading Agent
      │
      ├── research_agent(input)
      │       └── Research Agent ──→ search_financial_news()
      │
      ├── account_report() / lookup_share_price()
      ├── buy_shares() / sell_shares() / change_strategy()
      └── send_telegram_message()
```

{{% notice info %}}
The diagram shows one trader to avoid repeating the same structure. The runtime can execute one trader or all four traders; each trader creates its own Research Agent and uses it as a tool.
