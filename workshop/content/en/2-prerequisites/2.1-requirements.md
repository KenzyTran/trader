## Requirements before you begin

### Account and permissions

- An AWS account that does not use the root user for daily operations.
- AWS credentials configured through IAM Identity Center or a dedicated profile.
- Permissions to create AgentCore Runtime, IAM roles, CodeBuild, CloudWatch, and DynamoDB resources.
- Model access for the model configured in `MODEL_ID` in the selected Region.

### Tools

| Tool | Minimum version | Verify |
|---|---:|---|
| Python | 3.12 | `python --version` |
| uv | latest | `uv --version` |
| AWS CLI | v2 | `aws --version` |
| Node.js | 20 | `node --version` |
| AgentCore CLI | latest | `agentcore --version` |
| Hugo Extended | 0.123.8 | `hugo version` |

### Region

All resources and models must be in the same Region. This workshop uses `us-west-2` in its examples:

```powershell
aws configure set region us-west-2 --profile trader-workshop
$env:AWS_PROFILE="trader-workshop"
aws sts get-caller-identity
```

{{% notice warning %}}
Do not commit access keys, API keys, or Telegram bot tokens to Git. Store them only in a local `.env` file or use a credential provider or Secrets Manager in the cloud.
{{% /notice %}}
