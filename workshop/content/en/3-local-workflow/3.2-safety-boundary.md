## Boundary between the agent and executor

The target workflow uses structured output:

```text
TraderAgent
  └─ TradeProposal(action, symbol, quantity, rationale, confidence, risks)
       └─ PositionWeightRiskCheck
            ├─ rejected → store the reason; do not modify the account
            └─ approved → TradeExecutor → account state
```

The first risk rule only checks the maximum weight of a single position:

```text
projected_position_weight = projected_position_value / projected_portfolio_value
```

If the projected weight exceeds `MAX_POSITION_WEIGHT`, the proposal is rejected. The LLM is not responsible for interpreting this rule.

{{% notice info %}}
The risk workflow is developed incrementally according to `AGETNTS.md`. AWS deployment does not introduce additional risk rules beyond the agreed scope.
{{% /notice %}}
