## AgentCore entrypoint

The entrypoint is located at `deploy/agentcore/runtime.py` and uses `BedrockAgentCoreApp`:

```python
app = BedrockAgentCoreApp()

@app.entrypoint
async def invoke(payload):
    trader_name = payload.get("trader")
    return {"status": "completed", "traders": await run_traders(trader_name)}
```

A payload without `trader` runs all four agents; a payload containing a name runs only that trader.

### Create the AgentCore project

The AgentCore CLI manages deployment through a separate project. From the repository root:

```powershell
cd deploy/agentcore
agentcore create `
  --name TraderFloor `
  --framework Strands `
  --protocol HTTP `
  --model-provider Bedrock `
  --memory none `
  --build CodeZip
```

This creates a `TraderFloor` directory with CDK configuration and a sample agent. Synchronize the trader source and workshop entrypoint into the generated project:

```powershell
.\prepare_project.ps1
cd TraderFloor
```

Open `agentcore/.env.local` and add the variables from `../.env.example`. Then validate and run locally:

```powershell
agentcore validate
agentcore dev --logs
```

In another terminal:

```powershell
agentcore dev "Warren"
```

### Deploy to AWS

```powershell
agentcore deploy --plan
agentcore deploy
agentcore status
```

The AgentCore CLI packages the CodeZip, synthesizes CDK/CloudFormation, creates the IAM role and AgentCore Runtime, and returns the Runtime ARN.

{{% notice info %}}
CodeZip does not require Docker. Select Container only when the agent requires system dependencies or a custom image.
{{% /notice %}}
