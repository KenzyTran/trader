## Create the DynamoDB table

The project uses a single-table schema with partition key `pk` and sort key `sk`:

```powershell
aws dynamodb create-table `
  --table-name strands-trader-state `
  --attribute-definitions AttributeName=pk,AttributeType=S AttributeName=sk,AttributeType=S `
  --key-schema AttributeName=pk,KeyType=HASH AttributeName=sk,KeyType=RANGE `
  --billing-mode PAY_PER_REQUEST `
  --region us-west-2

aws dynamodb wait table-exists `
  --table-name strands-trader-state `
  --region us-west-2
```

### Runtime configuration

Create `.env.agentcore` from the example:

```powershell
Copy-Item deploy/agentcore/.env.example .env.agentcore
```

Important variables:

```dotenv
MODEL_PROVIDER=bedrock
MODEL_ID=us.anthropic.claude-sonnet-4-20250514-v1:0
AWS_REGION=us-west-2
STORAGE_BACKEND=dynamodb
DYNAMODB_TABLE=strands-trader-state
```

`MODEL_ID` is only an example. Use an inference profile or model enabled in your account and Region.

### Runtime execution role

At minimum, the Runtime role needs:

- `bedrock:InvokeModel` and `bedrock:InvokeModelWithResponseStream` for the selected model.
- `dynamodb:GetItem`, `PutItem`, `Query`, and `UpdateItem` on the workshop table.
- Permissions required by AgentCore to write CloudWatch logs and traces.

{{% notice warning %}}
The AgentCore CLI can create the role during the workshop. In a real environment, replace it with a least-privilege role scoped to specific resource ARNs.
{{% /notice %}}
